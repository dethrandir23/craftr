# Contributing to Craftr 🤝

First off, thank you for considering contributing to **Craftr**! Your help makes this project better for everyone.  

This guide will walk you through how to get started, report issues, submit code, and follow our coding standards.

---

## How to Contribute

### 1. Reporting Issues
If you find a bug, have a feature request, or notice something unclear:
- Check the [Issues](https://github.com/dethrandir23/craftr/issues) page to see if it’s already reported.
- If not, open a new issue with:
  - A descriptive title
  - Steps to reproduce (for bugs)
  - Expected vs. actual behavior
  - Screenshots or logs if applicable

### 2. Submitting Code

We love pull requests! Follow these steps:

1. **Fork the repository**  
   Click the **Fork** button at the top of the page.

2. **Clone your fork**  
   ```bash
   git clone https://github.com/YOUR_USERNAME/craftr.git
   cd craftr

    Create a feature branch

git checkout -b feature/amazing-feature

Make your changes

    Keep code clean and readable

    Write comments where necessary

    Follow the C++17 standard used in the project

Test your changes

mkdir build && cd build
cmake ..
make
./craftr --help

Commit and push

    git add .
    git commit -m "Add amazing feature"
    git push origin feature/amazing-feature

    Open a Pull Request
    Go to your fork on GitHub and click New Pull Request. Provide a clear description of your changes.

Coding Guidelines

    C++17 standard should be used

    Follow the project’s formatting style (similar to existing code)

    Keep functions short and modular

    Use meaningful variable and function names

    Add comments for complex logic

Code of Conduct

We expect everyone to follow a respectful and inclusive environment. Please see our Code of Conduct

for more details.
Need Help?

If you're unsure about anything, feel free to open an issue or contact the maintainers. We’re happy to help!

Thank you for helping make Craftr awesome! ❤️
