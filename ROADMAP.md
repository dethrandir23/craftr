# Craftr Roadmap

## Current Version
The current version of Craftr has evolved from a simple, hardcoded project starter into a flexible template-based system. Now you can:
- Use template files (`template.yaml`) to add and configure the files you want in your project.
- Customize content dynamically with replacers.
- Initialize projects with flexible and reusable templates.

---

## Short-Term Plans (Next Days)

### 1. Variables Refactor
- Extend variable system beyond basic ones (version, date, name, author).
- Allow users to define any custom variables for project templates.

### 2. License System Improvement
- Move licenses to a centralized `licenses` file/folder.
- Make it easy to add new license types.

### 3. Project & Template Structure Refactor
- Redesign project directory layout.
- Reorganize template folder structure for easier management and usage.

### 4. Packaging & Scripts
- Adjust paths and packaging system for release.
- Implement `config` and `first-run` scripts to streamline project setup.

### 5. Documentation Update
- Update `README.md` for the new version.
- Add detailed documentation for code usage and repository structure.

### 6. Debugging Features
- Add more debug options.
- Implement a debug mode for enhanced troubleshooting.

### 7. Code Style & Formatting
- Improve readability of the codebase.
- Standardize spacing, tabs, and new lines for consistency.

---

## Long-Term Vision
The aim is to evolve Craftr from a simple hardcoded tool into a fully flexible, template-driven project scaffolding tool, ready for public use and easy extension.
